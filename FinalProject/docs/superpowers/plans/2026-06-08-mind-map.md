# Mind Map Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build an editable mind map workflow that imports `#` heading text, creates normal VectorFlow nodes/connectors, supports plus-based child insertion, supports subtree collapse, and persists through `.vflow`.

**Architecture:** Add pure mind map logic in `src/core/mindmap.py` and keep UI wiring in `VectorFlowApp`. Extend existing shape and connector metadata so mind map relationships survive save/load without replacing the editor's shape model.

**Tech Stack:** Python 3, tkinter, unittest, existing VectorFlow `Document`, `FlowchartShape`, `ConnectorShape`, `ShapeStyle`, and `History`.

---

## File Structure

- Create `src/core/mindmap.py`: heading parser, tree model, layout, document fragment generation, metadata helpers, descendant/hidden-ID helpers, add-child helper.
- Create `tests/test_mindmap.py`: parser, generation, collapse, add-child, copy/paste metadata, app generation, and metadata round-trip tests.
- Modify `src/core/shapes.py`: add optional `metadata` dict to `FlowchartShape` and `ConnectorShape` serialization.
- Modify `src/core/document.py`: remap mind map metadata IDs during copy/paste.
- Modify `src/app.py`: add command bar/menu entry, import dialog, overlay controls, hit testing, add-child and collapse actions, redraw integration.
