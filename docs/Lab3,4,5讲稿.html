Lab3：Cohen-Sutherland 直线段裁剪

1. 这个实验解决什么问题

屏幕上有一个矩形裁剪窗口，一条线段可能完全在窗口里，也可能完全在窗口外，也可能一部分在里面、一部分在外面。裁剪算法要做的是只保留窗口里可见的部分。

课堂讲法

红色虚线只表示原始线段在窗口外的部分，绿色实线表示窗口内保留下来的可见部分。程序不是靠肉眼判断，而是先用端点区域编码算出裁剪结果，再在显示层把原始光栅化点按窗口内外分类，避免窗口内残留红色。

2. 核心算法：区域编码

把裁剪窗口四条边向外延长，平面会被分成 9 个区域。每个点用 4 位二进制表示它相对于窗口的位置。

左上
1001
上
1000
右上
1010
左
0001
内部
0000
右
0010
左下
0101
下
0100
右下
0110
端点编码情况 算法判断 怎么讲
两个端点都是 0000 简取 整条线在窗口内，直接保留。
两个端点编码按位与不为 0 简弃 两个端点在窗口同一侧外面，线段不可能穿过窗口。
其他情况 裁剪 找一个在外面的端点，求它和窗口边界的交点，替换端点后继续判断。

3. 手算例子

窗口：x 从 20 到 60，y 从 15 到 40；线段：P0=(10,20)，P1=(50,35)

P0 的 x=10 小于 xmin=20，所以 P0 在窗口左侧，编码是 0001。P1 在窗口范围内，编码是 0000。不能简取，也不能简弃，所以要求交点。

dx = 50 - 10 = 40
dy = 35 - 20 = 15

左边界 x = 20
比例 = (20 - 10) / 40 = 0.25
y = 20 + 0.25 * 15 = 23.75

所以 P0 被替换成 (20, 23.75)，新的线段两端都在窗口内，算法接受裁剪后的线段。

Lab4：Bezier 曲线绘制与变换

1. 这个实验解决什么问题

用户不需要直接给出曲线上的所有点，只需要给几个控制点，程序就能生成一条平滑曲线。控制点像拉住曲线的手柄，会影响曲线形状，但曲线不一定经过每个控制点。

课堂讲法

红色点是控制点，灰色线是控制多边形，蓝色像素是最终生成的 Bezier 曲线。拖动控制点时，曲线实时变化，这能直观看到控制点对曲线的影响。

2. 核心算法：De Casteljau 递推

De Casteljau 的核心就是反复做线性插值。给定参数 t，在相邻点之间按比例取点，一层一层递推到只剩一个点，这个点就是曲线在该 t 上的位置。

P_i^(0) = 原始控制点
P_i^(r) = (1 - t) * P_i^(r-1) + t * P_{i+1}^(r-1)
原始控制点 P0、P1、P2、P3
第一层插值 在相邻控制点之间按 t 取点
继续递推 新点之间继续插值
最后一个点 就是曲线上的采样点

3. 手算例子

控制点 P0=(0,0)，P1=(4,4)，P2=(8,0)，取 t=0.5 
第一层：
A = P0 和 P1 的中点 = (2, 2)
B = P1 和 P2 的中点 = (6, 2)

第二层：
C = A 和 B 的中点 = (4, 2)

所以 Bezier 曲线在 t=0.5 处的点就是 (4, 2)。

一句话解释

De Casteljau 不是直接展开复杂多项式，而是反复做“按比例取点”，所以几何意义直观，也比较稳定。

4. 曲线绘制、平移和旋转

曲线怎么画出来

de_casteljau() 只能算某一个 t 对应的点。要画整条曲线，代码把 t 从 0 到 1 分成 300 步，每个 t 算一个点，再把这些点四舍五入成像素。

t = 0/300, 1/300, 2/300, ..., 300/300

变换怎么做

平移就是每个控制点都加同样的 dx, dy。旋转是绕控制点质心旋转，质心就是所有控制点坐标的平均值。界面中的绿色十字就是旋转中心。

x' = x + dx
y' = y + dy

cx = 所有 x 的平均值
cy = 所有 y 的平均值

5. 代码对应

文件 函数或模块 讲解重点

Lab4/algorithms.py bezier_curve()、bezier_curve_pixels() 对 t 采样，生成曲线像素。
Lab4/algorithms.py translate_points()、rotate_points()、centroid() 实现平移、绕质心旋转和旋转中心计算。
Lab4/main.py 交互和左侧面板 加点、拖动、删点、平移、旋转，并显示阶数、坐标、质心和递推信息。

Lab5：三维形体、观察变换、消隐与光照

1. 这个实验解决什么问题

Lab5 从二维图形扩展到三维显示。程序构造了一个带墙体、屋顶和烟囱的小建筑模型，并实现物体旋转、摄像机漫游、透视投影、背面剔除、Z-buffer 消隐和简单 Phong 光照。

课堂讲法

Lab5 的重点不是用 pygame 直接画三维图形，而是自己完成三维渲染管线中的几个核心步骤。pygame 主要负责开窗口和把我算出来的像素显示出来。

2. 模型怎么表示

三维模型在代码里用“顶点 + 三角面”表示。顶点是 (x, y, z)，面用三个顶点索引表示，比如 (0, 1, 2) 表示一个三角形。

一个四边形会拆成两个三角形，因为三角形一定共面，计算光栅化、法向量和深度插值更稳定。

顶点：(-1.6, -1.0, -1.1)
三角面：(0, 1, 2)
四边形：拆成 (a, b, c) 和 (a, c, d)

3. 三维渲染主流程

模型顶点 小建筑的墙、屋顶和烟囱都由顶点和三角面组成。
变换和投影 物体旋转，转到摄像机坐标，再透视投影到屏幕。
剔除和光照 背面剔除减少不可见面，Phong 光照计算顶点颜色。
光栅化和消隐 三角形转成像素，用 Z-buffer 保留最近颜色。 
模型顶点
  -> 物体旋转和平移
  -> 转到摄像机坐标系
  -> 透视投影到屏幕
  -> 背面剔除
  -> 光照计算
  -> 三角形光栅化
  -> Z-buffer 消隐
  -> 显示像素

4. 摄像机和透视投影

摄像机用 position、yaw 和 pitch 表示。根据 yaw 和 pitch 可以算出 forward、right、up 三个方向向量。世界坐标点减去摄像机位置后，再和这三个方向做点积，就变成摄像机坐标。

screen_x = width / 2 + focal * x / z
screen_y = height / 2 - focal * y / z
投影例子

画布宽 580，高 520，焦距 430。一个点在摄像机坐标中是 (1, 0.5, 5)。

screen_x = 580/2 + 430*1/5 = 376
screen_y = 520/2 - 430*0.5/5 = 217

所以这个点会显示在屏幕坐标大约 (376, 217)。如果 z 变大，点会更靠近屏幕中心，这就是近大远小。

5. 背面剔除、Z-buffer 和光照

背面剔除

通过叉乘计算三角形法向量，再判断这个面是否朝向摄像机。背对摄像机的面直接跳过。

normal = (B - A) × (C - A)

Z-buffer 消隐

每个屏幕像素保存一个深度。新的三角形像素只有在更靠近摄像机时，才会更新颜色。

如果 new_z < old_z:
    更新颜色和深度
否则:
    不画这个像素
Z-buffer 例子

同一个屏幕像素上，三角形 A 的 z=5，三角形 B 的 z=2。因为 2 更小，B 更近，所以最后显示 B 的颜色。不管 A 和 B 谁先画，结果都应该保留最近的面。

Phong 光照一句话

环境光负责“看得见”，漫反射负责“面朝光源更亮”，高光负责“有一点反光质感”。代码按顶点算颜色，再在三角形内部插值，所以会有明暗过渡。

6. 三角形光栅化

Lab5 没有直接调用 pygame 的多边形填充，而是自己判断像素是否在三角形内部。核心是边函数：如果一个像素中心点在三角形三条边的同一侧，它就在三角形内部。

光栅化时还会用重心坐标插值 z 深度和三个顶点的颜色。z 深度用于 Z-buffer，颜色插值用于让三角形内部有自然过渡。

高频追问速查

Lab3 可能怎么问

为什么编码按位与不为 0 就能丢弃？

因为按位与不为 0 说明两个端点至少在窗口同一侧。比如两个点都在窗口左边，它们的编码都含有 LEFT 位，这条线段整体就在左侧，不可能穿过窗口。

Cohen-Sutherland 和中点画线法分别负责什么？

Cohen-Sutherland 负责算裁剪后的两个端点；中点画线法负责把这两个端点之间的线段离散成屏幕像素。

为什么现在窗口内不会残留红色？

之前如果先把整条原始线段画成红色，再重新用裁剪后的端点画绿色，两个线段的离散像素可能因为取整和斜率不同而没有完全重合。现在程序先对原始线段调用 midpoint_line() 得到同一批像素点，再用 split_points_by_clip_rect() 按窗口范围分类：窗口外点画红色虚线，窗口内点画绿色实线，因此不会依赖覆盖效果。

Lab4 可能怎么问

Bezier 曲线一定经过所有控制点吗？

不一定。它一定经过第一个和最后一个控制点，中间控制点主要负责影响曲线形状。

为什么用 De Casteljau？

它的几何意义直观，就是反复线性插值；数值上稳定，也天然支持任意阶数。

为什么旋转后坐标是小数？

旋转涉及 sin 和 cos，结果通常是浮点数。内部保留浮点数，绘制时再四舍五入到像素网格。

Lab5 可能怎么问

为什么模型要拆成三角形？

三角形一定共面，插值和光栅化简单稳定。复杂多边形通常也会先拆成三角形再渲染。

Z-buffer 和背面剔除有什么区别？

背面剔除是提前丢掉背对摄像机的面；Z-buffer 是像素级别处理前后遮挡。两个都朝向摄像机的面互相遮挡时，仍然需要 Z-buffer。

pygame 在 Lab5 中做了什么？

pygame 负责窗口、输入事件和最终像素显示。投影、光栅化、深度测试和光照颜色都是算法代码自己算的。
